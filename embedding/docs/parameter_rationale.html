파라미터 선택 근거

AIO 페르소나 분류 파이프라인 — 임베딩 전략·차원 축소·클러스터 수·샘플 크기·noise 임계값의 실험 근거

목차

  1. 임베딩 전략 선택 — percol5 (5120-dim)
  2. PCA 차원 선택 — n=100
  3. UMAP n_components 선택 — 2D
  4. KMeans k 선택 — k=5
  5. 샘플 크기 선택 — n=200,000
  6. noise_dist 임계값 선택 — 상위 10%
  7. Tier 3 PCA 차원 선택 — n=100 확정

Step 1 임베딩 전략 선택 — percol5 (5120-dim)

세 가지 임베딩 방식을 n=200,000, k=3~7 스윕으로 비교했다.

방식입력 칼럼dim best sil kbest sil 값 best ARI kbest ARI 값
embed (concat) 5칼럼 텍스트 concat → 단일 인코딩 1024 30.0205 40.640
percol (7col) 7칼럼 각각 독립 인코딩 → concat 7168 30.0116 30.631
percol5 ✓ 5칼럼 각각 독립 인코딩 → concat 5120 30.0142 50.537

bootstrap ARI는 같은 데이터에서 두 번 KMeans를 실행했을 때 클러스터 할당이 얼마나 재현되는지를 측정한다 (높을수록 안정적). percol5는 k=5에서 ARI 0.537로 세 방식 중 이론 목표(k=5)와 안정성이 동시에 가장 잘 일치했다.

concat 방식(embed)은 5개 칼럼의 신호가 평균화되어 특정 칼럼의 특성이 희석된다. percol5는 칼럼별 방향 정보를 독립 보존하면서 공간 크기를 percol 7col(7168-dim) 대비 28% 축소한다.

선택: percol5 (5120-dim) — k=5에서 bootstrap ARI 최고, 칼럼 신호 독립 보존, 7col 대비 공간 절약.

5120-dim → KMeans를 직접 적용하는 baseline 대비, PCA로 차원을 축소한 뒤 클러스터링하는 효과를 측정했다.

PCA-100이 bootstrap ARI 0.954로 모든 PCA 차원 중 최고이며, baseline(29.9s) 대비 KMeans 시간을 100배 단축한다. PCA-200은 ARI가 오히려 낮아지고 시간만 증가했다. UMAP 고차원 직접 사용은 ARI가 불안정하다 (n=10에서 0.579).

PCA는 클러스터링 전처리용이다. UMAP 2D로의 최종 투영은 §3에서 별도 수행한다 (L2→PCA(100)→L2→UMAP(2D) 파이프라인).
선택: PCA n=100 — bootstrap ARI 최고 (0.954), baseline 대비 100× 가속, PCA-200 대비 ARI 개선.

silhouette 0.584(n=2) vs 0.35(n≥5): KMeans는 볼록 클러스터를 가정하는데, UMAP 2D 공간에서 데이터가 볼록하게 압축되어 이 가정이 가장 잘 충족된다. 고차원에서는 거리 척도의 농도 현상(concentration of measure)이 발생해 클러스터 경계가 모호해진다.

선택: UMAP n_components=2 — silhouette 0.584로 압도적 1위, ARI 0.981, KMeans 볼록 가정 최대 충족.

Step 2 KMeans k 선택 — k=5

이론적 근거

Schwartz(2012) Basic Human Values의 10개 가치를 인접 쌍 5종으로 묶는 archetype 설계가 먼저 확정되었다. 소비태그 역시 소비심리학 라이프스타일 세분화 5종으로 설계했다. 따라서 k=5는 이론 요구사항이며, 실험은 k=5가 데이터에서도 지지되는지를 검증하는 역할을 한다.

실험 검증 (percol5, n=200,000)

ksilhouette ↑davies_bouldin ↓calinski_harabasz ↑bootstrap ARI ↑
30.01427.40032030.232
40.01227.70225200.443
5 ✓0.01197.24422110.537
60.00987.84818820.421
70.01097.05217190.404

silhouette는 k 증가에 따라 단조 감소한다 (고차원 임베딩의 특성). 클러스터 안정성 지표인 bootstrap ARI는 k=5에서 최고(0.537)로 이론값과 일치한다. k=6, 7에서 ARI가 하락하는 것은 클러스터가 과분할되어 재현성이 떨어짐을 의미한다.

UMAP 후 KMeans는 절대적 silhouette 값이 낮더라도 ARI 기준으로 안정적인 클러스터를 형성할 수 있다. 따라서 k 선택 기준을 silhouette 단독이 아닌 ARI 우선으로 삼았다.
선택: k=5 — 이론(Schwartz 5분위) + 실험(bootstrap ARI 최고 0.537) 동시 지지.

Step 2 샘플 크기 선택 — n=200,000

전체 데이터셋은 약 700만 건이다. 임베딩(9 GB parquet) 생성에 4~5시간이 소요되며, UMAP은 n에 비례하여 시간이 증가한다. n=10,000 / 50,000 / 200,000 세 규모에서 클러스터 품질을 비교했다.

npercol5 ARI (k=5)matrix 최대 셀matrix 최소 셀비고
10,000~0.45 (불안정)1,42897소수 클러스터 불안정
50,000~0.505,341327실험·검증용
200,000 ✓0.53720,2641,147최종 분석 기준

n=200,000에서 5×5 매트릭스의 최소 셀이 1,147건으로 통계적 해석에 충분하다. n=50,000은 개발·검증 단계에서 빠른 반복을 위해 병행 사용한다. 1M 전수는 UMAP 메모리 한계(RAM ~60 GB 필요)로 현재 하드웨어에서 실행 불가하다.

선택: n=200,000 — bootstrap ARI 안정화, 최소 셀 1,147건 확보, UMAP 메모리 한계 내.

Step 2 noise_dist 임계값 — Shannon 엔트로피 상위 10%

각 샘플에서 5개 centroid까지의 역거리 softmax를 구해 Shannon 엔트로피 H를 계산한다. H가 높을수록 어느 클러스터에도 명확히 속하지 않는 경계 샘플이다.

noise_dist=1 판정 기준: H ≥ p90 (Shannon 엔트로피 상위 10%). 절대값 임계값 대신 상대적 분위수를 사용해 anchor 텍스트 버전이 바뀌어도 항상 전체의 10%가 플래그된다.

임계값noise_dist=1 비율유효 샘플(noise=0)평가
p80 (상위 20%)20%160,000과도한 제외
p90 (상위 10%) ✓10%180,000균형
p95 (상위 5%)5%190,000경계 샘플 일부 포함

noise_dist는 탈락이 아닌 플래그다. 분석 시 noise_dist=0 필터를 적용하면 n=180,000의 고신뢰 샘플만 사용할 수 있고, 전체 포함 시와 비교해 결과의 강건성을 확인할 수 있다 (matrix_5x5_n{N}.csv vs matrix_5x5_all_n{N}.csv).

선택: p90 (상위 10%) — 180,000건 유효 샘플 유지, 재현 가능한 상대적 기준, --noise-pct로 조정 가능.
